# Deploying the demo to Hugging Face Spaces

A Space is its own git repository on huggingface.co -- you don't push this
whole research repo there, just app.py, src/, data/sample_articles.db, and
two renamed files. Steps below assume you have a Hugging Face account.

## 1. Create the Space

Go to https://huggingface.co/new-space and create a Space:
- SDK: **Gradio**
- Hardware: **CPU basic** (free tier) -- do NOT pick a ZeroGPU template.
  This demo doesn't use a GPU, and ZeroGPU hardware requires an
  `@spaces.GPU`-decorated function to even start (see Troubleshooting
  below for the fix if you picked it, or already have a ZeroGPU Space).
- Visibility: your choice

This gives you a git remote URL like
`https://huggingface.co/spaces/<your-username>/narrative-drift-detection`.

## 2. Set the HF_TOKEN secret

The LLM-based stages (latent frame labeling, actor-frame graph) call the
Hugging Face Inference API, which needs your own token:
1. Create a token at https://huggingface.co/settings/tokens with "Make calls
   to Inference Providers" permission.
2. In the Space's Settings -> Variables and secrets, add a secret named
   `HF_TOKEN` with that token's value.

(Optional) You can also set a `HF_INFERENCE_MODEL` variable there to
override the default model (`Qwen/Qwen2.5-7B-Instruct`) if you'd rather use
a different one available on the Inference API.

## 3. Push the Space files

From a scratch directory next to this repo:

```bash
git clone https://huggingface.co/spaces/<your-username>/narrative-drift-detection space
cd space

cp -r ../ProiectDisertatie/src .
cp -r ../ProiectDisertatie/data .
cp ../ProiectDisertatie/app.py .
cp ../ProiectDisertatie/deploy/requirements-space.txt requirements.txt
cp ../ProiectDisertatie/deploy/SPACE_README.md README.md

git add -A
git commit -m "Deploy narrative drift detection demo"
git push
```

The Space builds automatically (installing requirements.txt, including
downloading the spaCy model) -- the first build typically takes a few
minutes because of the sentence-transformers/spaCy/torch downloads.

## 4. Verify

Once the build finishes, open the Space URL, pick a source, and click
"Run analysis". Try it once with the LLM checkbox off first (fast, no
external calls) and once with it on (exercises the Inference API call and
needs the HF_TOKEN secret from step 2).

## Troubleshooting

Things that actually went wrong deploying this once, in the order they
surfaced -- `app.py` already contains the fixes below, so a fresh deploy
following steps 1-3 shouldn't hit them again. Listed in case a future
change reintroduces one, or a similar Space runs into the same issues:

- **`BUILD_ERROR`, a `.db` file "should have been a pointer, but wasn't"**:
  some local SQLite file (e.g. a stray `src/frame_cache.db` generated by
  running the pipeline locally) got copied into the Space's `src/` and
  committed as a raw blob despite `.gitattributes` marking `*.db` for Git
  LFS (this happens if `git lfs install` wasn't run before committing).
  Fix: don't copy locally-generated `.db` files into the Space; if one
  slips in, `git rm --cached <path>` it.
- **`BUILD_ERROR`, `pip` can't find `pandas==3.0.2` (or similar)**: the
  Space's Docker image runs Python 3.10 regardless of a `python_version`
  field in the README frontmatter -- that field did not change the actual
  base image in practice. Fix: don't pin exact versions in
  `requirements.txt` for packages that move fast (pandas/numpy/spacy/...);
  let pip resolve whatever's compatible with 3.10.
- **`RUNTIME_ERROR`, log ends at "Stopping Node.js server..."**: Gradio 6
  defaults to server-side rendering via a Node.js proxy, which doesn't
  come up cleanly on this Space. Fix: `demo.launch(ssr_mode=False)`.
- **`RUNTIME_ERROR`, "CUDA has been initialized before importing the
  `spaces` package"**: Gradio auto-spawns a background thread that
  imports the `spaces` package (always present in HF's Docker image) for
  hot-reload support; if `torch` (pulled in transitively by
  sentence-transformers) gets imported first, that thread crashes. Fix:
  `import spaces` as the very first thing in `app.py`, guarded with
  `try/except ImportError` since `spaces` isn't installed outside a Space.
- **`RUNTIME_ERROR`, `errorMessage: "No @spaces.GPU function detected
  during startup"`** (only visible via the API's `info.runtime.raw`, not
  in the build/run logs): the Space is provisioned with ZeroGPU hardware,
  which refuses to start without at least one `@spaces.GPU`-decorated
  function, even though nothing here needs a GPU. Ideally: change the
  Space's hardware to `cpu-basic` in Settings (or
  `HfApi().request_space_hardware(repo_id, hardware="cpu-basic")` --
  note this specific downgrade returned "402 Payment Required" without a
  PRO subscription when tried here). Workaround used instead: a dummy
  `@spaces.GPU`-decorated function that's never actually called, just
  present so the startup check passes.

If a Space gets stuck, `HfApi().space_info(repo_id).runtime.raw` surfaces
the real `errorMessage` far more often than the build/run log streams do.

## Notes for whoever reviews this (e.g. the professor)

- This demo intentionally runs on a small, truncated sample
  (`data/sample_articles.db`, ~1,400 articles, 800-character excerpts,
  Jan-Dec 2020 COVID-19 coverage, BBC/CNN/The Guardian/The Washington
  Post) rather than the full ~40,000-article corpus, to avoid
  redistributing scraped news article text publicly. See
  `scripts/build_sample_dataset.py` for how it was built and why that
  particular topic/window/source combination was chosen (each source
  has real gaps in different topics/periods in the underlying data).
- The full pipeline (24 analysis stages: narrative signatures/archetypes,
  cross-source divergence, change-point detection, evidence retrieval, and
  more, versus the ~6 shown in the demo) runs locally against the complete
  corpus -- see the main `README.md` and `src/main.py`.
- The demo's LLM-based stages use the Hugging Face Inference API; the
  dissertation's own experiments use a local Ollama server. Both share the
  same code, switched via `LLM_BACKEND` -- see `src/llm_backend.py`.
